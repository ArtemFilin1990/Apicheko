# Codex agent instructions

1. Перед изменениями прочитай `AGENTS.md` в текущей области и релевантные `README.md`/конфиги.
2. Делай минимальные, обратимые изменения без лишнего рефакторинга.
3. Не добавляй секреты в код, тесты, логи и документацию.
4. Не глуши ошибки интеграций: сервисные ошибки должны быть явными для пользователя и диагностики.
5. Для Worker-потока сначала проверяй `worker/worker.js` и актуальные Checko endpoint-мэппинги.

## Автопроверки (запускать из корня репозитория)

- `node --check worker/worker.js`
- `node --test tests/worker_smoke.test.mjs`
- `python -m unittest discover -s tests -p "test_*.py"`

Если проверка неприменима к изменению или окружению, зафиксируй причину в отчёте.
